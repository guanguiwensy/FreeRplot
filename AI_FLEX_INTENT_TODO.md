# AI 灵活意图执行改造 TODO（替代“预设命令触发”）

## 目标
- 用户无需记命令模板，使用自然语言即可改图。
- AI 能理解“我要什么效果”，并转换成可执行 patch。
- 不确定时给出最小澄清，而不是静默失败。

## 当前问题（必须解决）
- [ ] 依赖硬编码短语匹配（如“标题改成...”）
- [ ] 触发条件窄，表达稍有变化就失效
- [ ] 缺少“解析失败后的澄清流程”
- [ ] 缺少“执行预览与确认”机制

## 设计原则
- [ ] 不让用户背命令：支持口语、同义表达、上下文省略
- [ ] 先理解意图，再映射参数，不反过来
- [ ] 可解释：告诉用户“系统理解成了什么、改了什么”
- [ ] 安全兜底：低置信度不直接改，先确认

## Phase 1：意图层重构（核心）
- [ ] 新增 `intent_engine`（独立函数/文件）
- [ ] 定义统一意图 schema：
  - [ ] `intent_type`（modify_current / recommend_new / explain / unknown）
  - [ ] `targets`（title/x_label/y_label/theme/palette/range/option...）
  - [ ] `patch`（可直接执行的参数补丁）
  - [ ] `confidence`（high/medium/low）
  - [ ] `needs_clarification`（true/false）
  - [ ] `clarify_question`（澄清问题文本）
- [ ] 先走 LLM 结构化解析，再走本地规则兜底

## Phase 2：自然语言覆盖扩展（不靠固定句式）
- [ ] 同义词词典（中英混用）：
  - [ ] 标题/题目/title
  - [ ] 横轴/x轴/x label
  - [ ] 纵轴/y轴/y label
  - [ ] 透明度/alpha
  - [ ] 点大小/point size
  - [ ] 范围/区间/从…到…/between
- [ ] 数值表达解析：
  - [ ] “0到100”“0~100”“between 0 and 100”
  - [ ] 百分比、负数、小数
- [ ] 布尔表达解析：
  - [ ] 开启/关闭、显示/隐藏、要/不要

## Phase 3：执行策略升级
- [ ] 三段式执行：
  - [ ] 解析（理解用户意图）
  - [ ] 预览（展示即将修改字段）
  - [ ] 应用（写入控件并重绘）
- [ ] 高置信度：自动应用 + 回显变更
- [ ] 中置信度：先展示“我理解为…”让用户一键确认
- [ ] 低置信度：只提 1 个澄清问题（不刷屏）

## Phase 4：多轮上下文能力
- [ ] 支持省略主语的连续指令：
  - [ ] “再把透明度调低一点”
  - [ ] “标题也改成英文”
- [ ] 维护最近一次参数改动快照（用于“再/继续/恢复”）
- [ ] 增加撤销一步（undo last patch）

## Phase 5：冲突与校验
- [ ] 执行前参数合法性检查（范围、类型、上下界）
- [ ] 不可执行时返回可操作建议（不是失败）
- [ ] 对图表不支持的参数给出替代方案

## Phase 6：UI 交互改造
- [ ] 在 AI 回复区增加“理解结果卡片”
  - [ ] Intent 类型
  - [ ] 将修改字段
  - [ ] 置信度
- [ ] 增加按钮：
  - [ ] 应用
  - [ ] 重新解析
  - [ ] 撤销上一步

## Phase 7：评估与验收
- [ ] 建 30~50 条真实口语测试集（中文优先，含中英混说）
- [ ] 指标：
  - [ ] 意图识别准确率 >= 90%
  - [ ] patch 正确应用率 >= 85%
  - [ ] 需澄清比例 <= 20%
- [ ] 回归测试：不影响已有“图表推荐”流程

## 实施顺序（推荐）
1. 先做 Phase 1 + Phase 3（先可用）
2. 再做 Phase 2（提覆盖）
3. 再做 Phase 4 + Phase 5（提体验与稳定）
4. 最后做 Phase 6 + Phase 7（产品化验收）

## 对你这个反馈的直接修复目标
- [ ] “标题改成aaa”之外的表达也能执行，例如：
  - [ ] “把图名换成aaa”
  - [ ] “标题用aaa”
  - [ ] “title = aaa”
  - [ ] “给这张图起名aaa”
- [ ] 不再要求用户记命令格式。
